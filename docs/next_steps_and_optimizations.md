## 2. 优先级最高的待办事项

### 2.1 替换人口网格数据

**目标**：用 Eurostat/GISCO 人口网格替换当前模拟 `population_grids` 数据。

当前 `population_grids` 表是合成样例，只用于验证 SQL 流程。后续应下载 Eurostat/GISCO 2021 人口网格，并裁剪到比利时 + 荷兰。

建议步骤：

1. 下载 Eurostat/GISCO population grid。
2. 使用 QGIS 或 `ogr2ogr` 转换为 EPSG:3035。
3. 导入暂存表，例如 `stg_population_grid_2021`。
4. 清空或备份原 `population_grids`。
5. 将真实人口字段导入 `population_grids.population`。
6. 暂时保留 `edm_fan_index = 0.5`，并在报告中说明该字段仍为偏好模拟指标。

价值：

| 影响 | 说明 |
| :--- | :--- |
| 噪声影响人口估算 | 从演示结果变成真实数据支撑 |
| 候选地评分 | `population_score` 可基于真实人口密度计算 |
| 报告可信度 | 明显提高空间分析部分的严谨性 |

### 2.2 重新计算候选地评分

**目标**：用真实空间数据动态计算 `site_evaluations`，替换当前手工演示评分。

当前 `site_evaluations` 中的评分是手工设定，用于演示模型。后续应基于以下真实数据重新计算：

| 评分项 | 建议计算方式 |
| :--- | :--- |
| `airport_score` | 候选地到最近机场距离，距离越近分越高，但可设置过近惩罚 |
| `population_score` | 候选地 50 km 范围内人口或加权人口 |
| `ecology_safety_score` | 是否与 Natura 2000 相交、距离保护区远近 |
| `noise_safety_score` | 3 km 或 5 km 范围内高敏感设施数量和最近距离 |
| `total_score` | 使用计划中的加权公式计算 |

建议公式仍采用：

```text
total_score =
    0.30 * airport_score
  + 0.25 * population_score
  + 0.25 * noise_safety_score
  + 0.20 * ecology_safety_score
```

建议新增脚本：

```text
sql/07_recalculate_site_scores.sql
```

价值：

| 影响 | 说明 |
| :--- | :--- |
| 模型真实性 | 从“手工评分”升级为“PostGIS 计算评分” |
| 结果可解释性 | 每个分项都能追溯到空间查询 |
| 报告亮点 | 能展示完整的选址评价流程 |

---

---

### 4.2 清洗候选地数据

当前 `candidate_sites` 已经包含 OSM 候选地，但仍需要进一步清洗。

建议清洗规则：

1. 面积小于 50,000 平方米的候选地不进入正式分析。
2. 排除明显不适合音乐节的城市小公园、纪念公园、封闭设施。
3. 对露营地、公园、活动场地分别设置不同权重。
4. 删除重复或重叠度极高的候选地。
5. 对候选地名称为空的数据进行人工补充或剔除。

优先级：中高。

---

### 4.3 优化 Natura 2000 几何质量

导入 Natura 2000 时，PostGIS 提示少量几何存在 ring self-intersection。

当前处理方式：

```sql
WHERE ST_IsValid(geom)
```

后续优化方式：

```sql
ST_MakeValid(geom)
```

建议：

1. 新增清洗脚本 `sql/08_clean_geometries.sql`。
2. 对保护区几何使用 `ST_MakeValid`。
3. 对候选地几何也统一做一次合法性检查。

优先级：中。
